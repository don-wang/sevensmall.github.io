WHAT?

A mere website of a mere human here, recording some of my ideas and thoughts.

WHY?

Once I built a blog with Wordpress when learning PHP. To me it's so fun to make websites, but kind of boring to update the contents. Since I'm lazy and not good at writing posts, that blog was laid aside and abandoned after a short while.

Now we got another guinea pig. Recently I started learning Python and Flask. While reading some references article I encountered Pelican, a static blog engine based on python.

The simple reason why I take to Pelican is simple. It is simple to learn, simple to write, simple to deploy. So why not start a blog with it?

Actually, the deadline of my Master Thesis is two weeks later. But I didn't start writing yet. I think perhaps write something on blog first would be helpful to the thesis process :'-(

HOW?

There are not too much required knowledge for Pelican. Some basic python and HTML skill should be enough.

Install

You can start your local service by only 3 steps:

$ pip install pelican   # install pelican via pip
$ pelican-quickstart    # setup with prompt questions
$ make devserver        # start a development web server

Then you can access your pelican at http://localhost:8000. To find more detail instructions, many resources are available, like the official Guideline is very easy and clear to follow.

Theme

After the basic blog was set up, I need some customize especially the theme. Keeping the simple style of Pelican, I found the this SVTLE RESPONSIVE. Now you have seen the result of modification.

Server Host

Unlike Wordpress or other blog systems, Pelican only need to show static files, thus the hosting could have many options.

GitHub Pages is one very good choice: Free, Trusty, Simple.
